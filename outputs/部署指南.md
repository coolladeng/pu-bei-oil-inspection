# 石油行业巡检与设备管理系统 — 部署指南

## 环境要求

| 组件 | 版本要求 |
|------|---------|
| Docker | 20.10+ |
| Docker Compose | v2.0+ |
| 内存 | 最低 4GB，推荐 8GB |
| 磁盘 | 最低 20GB（含数据库和上传文件） |

## 快速部署 (Docker Compose)

### 1. 配置环境变量
```bash
cp .env.example .env
# 编辑 .env，修改 MYSQL_ROOT_PASSWORD 和 SECRET_KEY
```

### 2. 启动所有服务
```bash
docker compose up -d
```

服务启动顺序：MySQL → Redis → Backend → Frontend (Nginx)

### 3. 验证服务
```bash
# 检查服务状态
docker compose ps

# 测试后端健康检查
curl http://localhost:8000/api/health

# 前端访问
# 浏览器打开 http://localhost
```

### 4. 初始化数据库
首次部署后，后端会自动创建数据库表。如需导入初始数据：
```bash
docker compose exec backend python -c "from app.database import init_db; import asyncio; asyncio.run(init_db())"
```

## 服务端口说明

| 服务 | 端口 | 说明 |
|------|------|------|
| Nginx (前端) | 80, 443 | Web 访问入口，反向代理后端 API |
| FastAPI (后端) | 8000 | API 服务，仅内网访问 |
| MySQL | 3306 | 数据库，仅内网访问 |
| Redis | 6379 | 缓存/会话，仅内网访问 |

## 手动部署 (无 Docker)

### 后端
```bash
cd backend
pip install -r requirements.txt
# 配置 .env 文件
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

### 前端
```bash
cd frontend
npm install
npm run build
# 将 dist/ 部署到 Nginx/Apache
```

### 系统依赖
- FFmpeg: 视频时长检测和压缩
- MySQL 8.0: 主数据库
- Redis 7: 缓存和 WebSocket 会话

## 定时任务

后端内置 APScheduler，启动后自动执行：
- 月度巡检计划生成：每月1日 02:00
- 超时/漏检检测：每小时30分
- 任务截止日期检查：每日 06:00
- 设备状态告警：每日 08:00
- 隐患积压提醒：每日 08:05

## 备份策略

### 数据库备份
```bash
docker compose exec mysql mysqldump -u root -p oil_inspection > backup_$(date +%Y%m%d).sql
```

### 上传文件备份
```bash
docker compose cp backend:/app/uploads ./uploads_backup/
```

## 故障排查

### 数据库连接失败
- 检查 MySQL 是否就绪：`docker compose logs mysql`
- 确认 .env 中密码正确

### 文件上传失败
- 检查 uploads 目录权限
- 确认 FFmpeg 已安装：`docker compose exec backend ffmpeg -version`

### WebSocket 连接失败
- 检查 Nginx 配置中 proxy_read_timeout 设置
- 检查防火墙是否开放对应端口
