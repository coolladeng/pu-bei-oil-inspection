石油行业巡检与设备管理系统 - 项目方案 v3.0 (终稿)

========================================
更新说明
========================================
- 取消语音转文字功能，改为纯文字描述
- 视频上传：单条隐患最多3个，每个不超过50MB，时长不超过60秒

========================================
一、组织架构设计
========================================

公司（超级管理员/系统管理员）
 |
 ├── 大队A（大队级管理员）
 |    ├── 班组1（班组长）
 |    |    ├── 岗位员工A（巡检员）
 |    |    ├── 岗位员工B（巡检员）
 |    |    └── 岗位员工C（设备维保员）
 |    └── 班组2（班组长）
 |         └── 岗位员工D
 |
 └── 大队B（大队级管理员）
      └── 班组3
           └── 岗位员工E

角色权限矩阵:
| 角色 | 权限范围 |
|------|---------|
| 公司管理员 | 全部权限，跨大队查看 |
| 大队管理员 | 本大队全部权限 |
| 班组长 | 本班组成员管理 + 本班组巡检监督 |
| 巡检员 | 巡检执行 + 隐患上报 |
| 设备维保员 | 设备检查 + 隐患处理 |
| 调度员 | 大屏查看 + 报表查看（只读） |
| 审核员 | 隐患审核 + 验收（只读统计） |

========================================
二、功能模块详细设计 v3.0
========================================

--- 模块一：移动端NFC巡检系统 ---

1) 巡检点管理（PC端配置）
   - 巡检点名称、编号、NFC标签UID、GPS坐标
   - 所属大队、所属班组（三级联动）
   - 关联岗位/班组
   - 状态：启用/停用

2) 班组管理（PC端配置）
   - 班组名称、所属大队
   - 班组长分配
   - 班组成员列表（巡检员/维保员）
   - 班次：白班/中班/夜班/24h

3) 巡检规则配置
   - 检查时间窗：+-30分钟
   - 超时判定：超出时间窗60分钟
   - 提前检查限制：早于30分钟
   - 漏检判定：23:59:59
   - 月度计划一键生成

4) 巡检员操作界面
   - 今日待巡检列表（按班组/区域分组）
   - NFC自动打卡识别
   - 拍照（时间+GPS水印）
   - 隐患上报（拍照+视频+文字描述）

5) 隐患上报闭环流程（四步）
   上报(巡检员) → 审核(审核员/班组长) → 处理(维保员) → 验收(审核员)
   - 拍照：最多5张
   - 视频：最多3个，每个≤50MB，≤60秒
   - 文字描述：手动输入
   - 紧急程度：一般🟡/重要🟠/紧急🔴

6) 离线支持
   - 离线缓存巡检记录
   - 离线NFC可用
   - 联网自动同步

--- 模块二：移动端设备检查模块 ---

1) 设备管理（PC端）
   - 设备档案：编号/名称/型号/厂家/位置/大队/班组
   - 运行参数配置（温度/压力/转速范围）
   - 二维码生成与打印
   - 检查项配置（单选/数值/文本/拍照）

2) 设备扫码检查流程
   扫码 → 加载设备和检查任务 → 逐项检查 → 异常转隐患 → 提交

3) 检查项类型
   - 单选：正常/异常/需清洁
   - 数值：输入值+正常范围+单位
   - 文本：自由描述
   - 拍照：必须上传

--- 模块三：PC端调度室实时监控大屏 ---

1) 大屏布局（深色科技风）
   - 选择大队/班组（可切换）
   - 站点平面图 + 巡检点标注
   - 巡检进度：完成率/已检/未检/超时/提前
   - 各班组进度条排名
   - 设备状态总览
   - 实时告警滚动
   - 今日统计

2) 巡检监控页
   - 各班组/岗位完成情况
   - 未完成明细：班组+员工+巡检点+状态
   - 异常巡检记录（含照片）

--- 模块四：PC端后台管理系统 ---

1) 仪表盘
   - 概览数字（用户/巡检点/设备/今日隐患）
   - 今日待办：待审核/待处理/超时预警
   - 各大队巡检完成率对比

2) 巡检管理
   - 巡检点：CRUD + 批量导入 + NFC绑定 + 地图标注
   - 班组：CRUD + 人员分配 + 班次设置
   - 巡检计划：月度生成 + 日历视图 + 执行查看
   - 巡检监督：实时进度 + 预警 + 统计报表

3) 设备管理
   - 设备CRUD + Excel导入
   - 二维码生成打印
   - 检查项配置（动态添加）
   - 维保管理（周期+提醒+记录）

4) 检查/维保任务
   - 手动创建任务：指定班组/人员/检查项/频率/截止
   - Excel批量导入
   - 执行跟踪：当日完成情况 + 未完成明细

5) 隐患管理
   - 隐患列表 + 状态流转
   - 隐患审核/处理/验收操作
   - 隐患统计 + 趋势分析

6) 统计分析
   - 巡检：完成率/效率排名/漏检排行
   - 设备：完好率/故障率/维保及时率
   - 隐患：类型占比/趋势/处理时长
   - 导出：Excel/PDF

7) 组织管理
   - 大队管理：增删改查
   - 班组管理：关联大队 + 班组长分配
   - 用户管理：关联班组 + 角色分配
   - 部门树形结构展示

8) 系统配置
   - 巡检规则阈值
   - 通知规则（站内消息）
   - 数据保留期限

三、技术选型 v3.0
========================================

移动端(Android):
- Kotlin + Jetpack Compose
- Android NFC API (NTAG213)
- CameraX + ML Kit
- Retrofit + OkHttp
- Room (离线缓存)

- FFmpeg (视频压缩)

PC端:
- Vue 3 + TypeScript
- Element Plus
- ECharts 5
- WebSocket
- Vite
- Apache HTTP Server

四、数据库设计 v3.0
========================================

核心变更：增加组织架构表

1. sys_dept          - 部门/大队/班组 三级树形结构
2. sys_user          - 用户（关联dept_id + role_id）
3. sys_role          - 角色
4. sys_position      - 岗位
5. user_position     - 用户岗位关联
6. run_point         - 巡检点（关联dept_id）
7. run_point_pos     - 巡检点岗位关联
8. run_plan          - 巡检计划
9. run_record        - 巡检记录
10. equipment        - 设备（关联dept_id）
11. equip_check_item - 设备检查项
12. check_task       - 检查任务（关联班组dept_id）
13. check_task_equip - 任务设备关联
14. check_result     - 检查结果
15. hazard           - 隐患
16. hazard_flow      - 隐患流转
17. hazard_attachment- 隐患附件
18. sys_config       - 系统配置

关键变更说明:
- 巡检点/设备增加 dept_id 字段，支持按大队/班组维度查看
- 检查任务关联 dept_id（班组），由班组成员自动接收
- 用户表 dept_id 指向用户所属班组
- 巡检计划按班组+巡检点+日期生成
- 监督报表可按大队→班组→个人三级下钻

五、API接口概要
========================================

认证: POST /api/auth/login, GET /api/auth/profile
组织: GET/POST/PUT/DELETE /api/departments
      GET/POST/PUT/DELETE /api/users
      GET/POST/PUT/DELETE /api/roles
巡检点: GET/POST/PUT/DELETE /api/run-points
岗位: GET/POST/PUT/DELETE /api/positions
巡检计划: GET /api/run-plans, POST /api/run-plans/generate-monthly
巡检记录: POST /api/run-records
设备: GET/POST/PUT /api/equipments, POST /api/equipments/import
      GET /api/equipments/{id}/qrcode
检查项: GET/POST /api/equipments/{id}/check-items
检查任务: GET/POST/PUT /api/check-tasks, POST /api/check-tasks/import
检查结果: POST /api/check-results
隐患: POST /api/hazards, PUT /api/hazards/{id}/review/handle/accept
统计: GET /api/stats/daily-run, /api/stats/position-run
      GET /api/stats/equipment, /api/stats/hazard
WebSocket: run_progress, hazard_alert, equipment_status, task_complete

六、开发阶段规划 v3.0（单人+AI）
========================================

考虑到单人开发+AI助手，建议压缩以下优先级:

Phase 0: 环境搭建 + 技术验证（3-5天）
- 本地开发环境搭建（Python/Node/Apache/MySQL）
- 移动端HelloWorld + NFC读取测试
- 确定技术栈可行性

Phase 1: 基础框架 + 组织权限（1-2周）
- FastAPI项目骨架
- MySQL建表（含组织架构）
- JWT认证 + RBAC
- Vue3项目骨架 + 登录页 + 菜单框架
- Android项目骨架 + 登录页
- Apache反向代理配置

Phase 2: 组织管理 + 巡检模块（2-3周）
- 后端：部门/班组/用户管理API
- 后端：巡检点管理API + NFC绑定
- 后端：岗位管理 + 月度计划生成
- 后端：巡检记录API + 定时任务
- PC端：组织管理页面
- PC端：巡检管理页面（巡检点/岗位/计划/监督）
- 移动端：登录 + 今日待巡检界面
- 移动端：NFC打卡 + 巡检提交

Phase 3: 设备检查模块（1-2周）
- 后端：设备管理API + 二维码生成 + Excel导入
- 后端：检查项 + 检查任务 + 检查结果API
- PC端：设备管理页面 + 任务配置
- 移动端：扫码检查界面

Phase 4: 隐患管理闭环（1-2周）
- 后端：隐患上报 + 审核/处理/验收API
- 后端：文件上传（照片+视频）+ 视频压缩
- PC端：隐患管理页面 + 审核/处理/验收操作
- 移动端：隐患上报界面

Phase 5: 调度室大屏 + 统计（1-2周）
- 后端：WebSocket推送 + 统计聚合API
- PC端：实时监控大屏（ECharts）
- PC端：巡检/设备/隐患统计报表
- PC端：报表导出

Phase 6: 优化 + 部署（1周）
- 离线缓存完善
- 性能优化
- 部署到服务器
- 用户手册

总工期预估：8-12周（单人+AI）
