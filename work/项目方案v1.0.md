# 石油行业巡检与设备管理系统 - 项目方案 v1.0

## 一、项目概述

### 1.1 项目背景
面向石油行业（采油、炼化、储运等场景），构建一套覆盖"移动端巡检 + 设备检查 + PC调度室大屏"的综合性管理平台，实现：
- 巡检过程数字化、可追溯
- 设备维保状态实时掌握
- 调度室对全站点运行状态的可视化监控
- 隐患发现→上报→处理的闭环管理

### 1.2 核心用户角色
| 角色 | 说明 |
|------|------|
| **超级管理员** | 系统全局配置、用户管理、角色权限分配 |
| **初级管理员** | 巡检计划配置、巡检点管理、巡检监督（漏检/超时/提前检查判定） |
| **巡检员** | 日常巡检执行、隐患上报、NFC打卡 |
| **设备检查员/维保人员** | 设备扫码检查、维护保养操作 |
| **调度室值班员** | 大屏监控、当日状态查看、异常处理跟踪 |

## 三、功能模块详细设计

### 模块一：移动端 - NFC 巡检系统

#### 3.1.1 巡检点管理（初级管理员）
- 创建/编辑/删除巡检点（名称、位置坐标、关联NFC标签ID、所属区域）
- 批量导入巡检点（Excel模板）
- NFC标签绑定与解绑
- 巡检点状态管理（启用/停用）

#### 3.1.2 巡检岗位管理
- 创建巡检岗位（岗位名称、班次类型：早/中/晚/24h）
- 岗位关联人员（可多人共用一岗位）
- 岗位关联巡检点（一个岗位覆盖一组巡检点）
- 岗位工作时间段配置

#### 3.1.3 巡检规则配置
| 规则项 | 说明 |
|--------|------|
| 检查时间窗 | 每个巡检点的允许检查时间段 |
| 巡检周期 | 每日/每周/每月/自定义间隔 |
| 超时判定 | 超出时间窗后X分钟标记为"超时" |
| 提前检查 | 早于时间窗前X分钟判定为"提前"（防补签） |
| 漏检判定 | 到当日23:59仍未检查标记为"漏检" |
| 月度计划 | 一键生成整月巡检计划，支持按周/日重复 |

#### 3.1.4 巡检员操作界面
- **今日待巡检**：展示当日所有待检查点列表
  - 显示：巡检点名称、位置、计划时间、NFC距离/已扫描
  - 按区域/岗位分组
  - 未检/已检/超时/提前 状态颜色标识
- **NFC打卡**：
  - 靠近巡检点NFC标签自动识别
  - 支持手动输入NFC ID备用
  - 打卡时自动记录：时间、位置GPS、用户ID、设备信息
- **拍照记录**：
  - 每个巡检点可配置是否必须拍照
  - 支持多张照片 + 文字备注
  - 照片自动添加时间水印和GPS水印
- **隐患上报**：
  - 发现异常时一键切换"隐患上报"模式
  - 拍照 + 选择隐患类型（安全/设备/环境/其他）
  - 选择紧急程度（一般/重要/紧急）
  - 自动关联巡检点 + 位置信息

#### 3.1.5 离线支持
- 网络不佳时本地缓存巡检记录
- 恢复网络后自动同步到服务器
- 离线期间仍可使用NFC功能

### 模块二：移动端 - 设备检查模块

#### 3.2.1 设备二维码管理
- PC端为每台设备生成唯一二维码（包含设备ID）
- 支持批量打印二维码标签
- 二维码内容：`https://[域名]/device/{deviceId}` 或直接设备ID

#### 3.2.2 设备扫码检查流程
```
扫描设备二维码
    → 获取设备基本信息（名称、编号、型号、安装位置）
    → 显示该设备的检查任务列表（根据岗位/频率匹配）
    → 逐项检查：正常/异常/需维保
    → 异常时自动跳转隐患上报流程
    → 提交后记录到服务器
    → PC端实时可见
```

#### 3.2.3 设备档案
每台设备记录：
- 基本信息：编号、名称、型号、厂家、安装日期、位置
- 技术参数：运行参数范围（温度、压力、转速等，供巡检时参考）
- 检查项目：关联的检查项列表
- 维保记录：历史维护保养记录时间线
- 当前状态：正常/异常/待维保/维修中

### 模块三：PC端 - 调度室实时监控大屏

#### 3.3.1 实时大屏显示
- **站点地图/平面图**：标注所有巡检点、设备位置
- **实时状态面板**：
  - 今日巡检进度：已检/未检/超时/提前（数字 + 进度条）
  - 各岗位巡检完成率排名
  - 当前在线巡检员人数
  - 实时隐患告警滚动列表
- **告警中心**：
  - 最新隐患报警（红/黄/橙色分级）
  - 报警位置、类型、处理状态
  - 一键推送处理通知

#### 3.3.2 巡检监控统计
- **当日概览**：
  - 各岗位是否全部巡检完成（✓/✗ 标记）
  - 未完成巡检点明细：哪个岗位、哪位员工、哪个点
  - 异常巡检内容列表（带照片缩略图）
- **统计报表**：
  - 日/周/月巡检完成率
  - 漏检/超时/提前统计排行
  - 隐患趋势分析（折线图、饼图）
  - 设备故障率统计

### 模块四：PC端 - 后台管理系统

#### 3.4.1 设备管理
- 设备CRUD操作
- 批量导入设备（Excel模板：设备编号、名称、型号、参数等）
- 二维码生成与打印
- 设备检查项配置
  - 每个检查项：名称、检查标准、正常参数范围、检查方法
  - 支持勾选：正常/异常/NA
  - 支持上传检查标准附件（PDF/图片）

#### 3.4.2 检查/维保任务管理
- **创建检查任务**：
  - 开放式添加：用户自由填写检查内容
  - Excel批量导入检查任务
  - 选择目标岗位/人员
  - 设置检查频率（每日/每周/每月/自定义）
  - 设置截止时间
- **任务下发**：
  - 按日/周/月自动生成检查任务
  - 手动追加临时检查任务
- **执行跟踪**：
  - 当日各岗位检查完成情况一览
  - 未完成明细：岗位名称、员工姓名、未完成项
  - 查看某个未完成任务的详情
  - 逾期提醒（站内消息 + App推送）

#### 3.4.3 Excel导入功能
提供以下Excel模板下载：
- 巡检点导入模板
- 设备信息导入模板
- 检查项导入模板
- 检查任务导入模板
- 人员及岗位导入模板

#### 3.4.4 用户与权限管理
- 用户CRUD、部门/岗位树形结构
- 角色权限配置（RBAC模型）
  - 超级管理员：全部权限
  - 初级管理员：巡检配置、巡检监督
  - 巡检员：巡检执行、隐患上报
  - 调度员：大屏查看、报表查看
  - 维保人员：设备检查、维保记录

#### 3.4.5 系统配置
- 超时判定阈值配置
- 提前检查阈值配置
- 通知规则配置
- 数据保留周期

## 四、技术选型建议

### 4.1 移动端（Android）
| 技术 | 选型 | 说明 |
|------|------|------|
| 开发语言 | Kotlin | 官方首选，NFC API支持完善 |
| UI框架 | Jetpack Compose | 现代声明式UI，科技感设计 |
| NFC | Android NFC API | 原生NFC读写支持 |

| 推送 | Firebase / 极光推送 | 告警通知 |
| 地图 | 高德地图SDK / 内置地图 | 位置信息 |

### 4.2 后端服务
| 技术 | 选型 | 说明 |
|------|------|------|
| 开发语言 | Java (Spring Boot) 或 Python (FastAPI) | Java适合大型企业级，Python开发更快 |
| API | RESTful + WebSocket | 常规接口 + 实时推送 |
| 数据库 | PostgreSQL | 关系型数据，JSON支持好 |
| 文件存储 | 本地/OSS | 照片、附件存储 |
| 缓存 | Redis | 在线状态、热点数据 |
| 任务调度 | XXL-JOB / Celery | 定时任务、超时判定 |
| Excel处理 | Apache POI / openpyxl | 导入导出 |
| 认证 | JWT | 无状态认证 |

### 4.3 PC端（Web）
| 技术 | 选型 | 说明 |
|------|------|------|
| 框架 | Vue 3 + TypeScript | 组件化开发，生态好 |
| UI框架 | Element Plus / Ant Design Vue | 企业级组件库 |
| 大屏可视化 | ECharts / DataV | 图表、实时大屏 |
| WebSocket | Socket.IO /原生WS | 实时同步 |
| 构建工具 | Vite | 快速开发 |

## 五、数据库核心表设计（概要）

### 核心实体关系
```
用户(User) ──< 岗位(Position) ──< 巡检计划(RunPlan) ──< 巡检记录(RunRecord)
                                    │
                                    ├─ 巡检点(RunPoint)
                                    │
                              设备设备(Equipment) ──< 检查任务(CheckTask) ──< 检查结果(CheckResult)
                                    │
                              检查项(CheckItem)
                                    │
                              隐患记录(Hazard)
```

### 关键表清单
| 表名 | 说明 |
|------|------|
| `sys_user` | 用户表（姓名、工号、手机号、密码、状态） |
| `sys_role` | 角色表 |
| `sys_position` | 岗位表 |
| `user_position` | 用户-岗位关联 |
| `run_point` | 巡检点表（名称、NFC_ID、GPS、区域） |
| `run_plan` | 巡检计划表（岗位、周期、时间窗、月份） |
| `run_record` | 巡检记录表（巡检点、用户、时间、状态、照片、备注） |
| `equipment` | 设备表（编号、名称、型号、二维码、参数） |
| `equipment_check_item` | 设备检查项表 |
| `check_task` | 检查/维保任务表 |
| `check_result` | 检查结果表 |
| `hazard` | 隐患记录表 |
| `sys_config` | 系统配置表 |

## 六、开发阶段规划

### Phase 1：基础框架 + 用户权限（2-3周）
- 后端：Spring Boot 项目搭建、数据库设计、用户认证、RBAC权限
- PC端：Vue3项目搭建、登录页、主页框架、菜单导航
- 移动端：Android项目搭建、登录页、主页框架

### Phase 2：巡检模块（3-4周）
- 后端：巡检点管理API、岗位管理API、巡检计划API
- PC端：巡检管理页面（巡检点、岗位、计划配置）
- 移动端：NFC巡检功能、今日待巡检界面、拍照上报
- 定时任务：超时/漏检/提前判定

### Phase 3：设备检查模块（2-3周）
- 后端：设备管理API、检查任务API、检查结果API
- PC端：设备管理页面、检查任务配置页面、Excel导入
- 移动端：扫码检查界面、检查结果提交

### Phase 4：调度室大屏 + 统计报表（2-3周）
- 后端：WebSocket实时推送、统计API、报表API
- PC端：实时监控大屏、巡检监控页、设备监控页
- 移动端：消息推送、通知中心

### Phase 5：优化 + 部署（1-2周）
- 离线缓存完善、性能优化
- 部署环境搭建、数据初始化
- 用户培训文档

## 七、界面风格建议

### 科技感设计要点
1. **主色调**：深蓝 (#0A1628) + 科技蓝 (#00D4FF) + 警示色（红/黄/绿）
2. **暗色主题**：大屏监控采用深色背景，突出数据可视化
3. **动效**：数据刷新时有平滑过渡动画，告警时有脉冲呼吸效果
4. **卡片式布局**：圆角卡片 + 半透明玻璃质感
5. **图标**：使用线性图标（如 Remix Icon / Material Icons）
6. **字体**：数据用等宽字体（JetBrains Mono）

## 八、待讨论事项

1. **NFC标签类型**：建议使用 NTAG213/NTAG215（成本低、兼容性好），还是 Mifare Classic？
2. **多站点支持**：是否需要支持多个石油站点/分场的独立管理？
3. **语音/视频上报**：隐患上报是否需要支持语音描述或短视频？
4. **与现有系统集成**：是否需要对接已有的OA系统或ERP系统？
5. **报表导出**：是否需要导出PDF格式的正式巡检报告？
6. **审批流程**：隐患上报是否需要经过"上报→审核→处理→验收"的审批流程？
7. **部署环境**：部署在内网服务器还是云服务器？是否有现有服务器资源？
8. **移动端型号**：需要适配哪些手机型号？是否使用工业平板？
