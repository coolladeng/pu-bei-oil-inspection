
========================================
四、技术选型
========================================

移动端(Android):
- Kotlin + Jetpack Compose
- Android NFC API (NTAG213/215)
- CameraX + ML Kit (扫码+拍照+录像)
- Retrofit + OkHttp (网络)
- Room (离线缓存)
- 百度/讯飞API (语音转文字)
- 极光推送 (告警通知)

后端服务:
- Python FastAPI (异步REST + WebSocket)
- MySQL 8.x (主数据库)
- Redis (缓存+会话)
- APScheduler (定时任务)
- openpyxl/pandas (Excel处理)
- PyJWT (认证)
- FFmpeg (视频压缩)

PC端(Web):
- Vue 3 + TypeScript
- Element Plus (UI组件库)
- ECharts 5 (可视化)
- WebSocket (实时推送)
- Vite (构建工具)
- Apache HTTP Server (反向代理+SSL+静态托管)

部署架构:
互联网 -> Apache(:443) -> FastAPI(:8000) -> MySQL + Redis + 文件存储

========================================
五、数据库核心表(MySQL)概要
========================================

核心表(15张):
1. sys_user          - 用户表(用户名/姓名/工号/手机/密码/状态/部门)
2. sys_role          - 角色表(角色名/编码/描述)
3. sys_position      - 岗位表(岗位名/班次/上下班时间/部门)
4. user_position     - 用户岗位关联表
5. sys_dept          - 部门表(部门名/上级部门)
6. run_point         - 巡检点表(编号/名称/NFC UID/GPS/区域/状态)
7. run_point_pos     - 巡检点岗位关联表
8. run_plan          - 巡检计划表(月份/岗位/巡检点/日期/时间窗/状态)
9. run_record        - 巡检记录表(计划ID/用户/巡检点/时间/数据/照片/NFC匹配/超时/提前)
10. equipment        - 设备表(编号/名称/型号/厂家/位置/二维码/状态/参数JSON)
11. equip_check_item - 设备检查项表(设备ID/项名/类型/正常范围/单位)
12. check_task       - 检查任务表(名称/类型/频率/岗位/检查项JSON/频率/截止)
13. check_task_equip - 任务设备关联表
14. check_result     - 检查结果表(任务/设备/用户/时间/结果JSON/照片)
15. hazard           - 隐患表(编号/类型/紧急度/上报人/位置/描述/照片/视频/状态)
16. hazard_flow      - 隐患流转记录表(隐患ID/步骤/操作人/动作/备注)
17. hazard_attachment- 隐患附件表(隐患ID/类型/文件URL/大小)
18. sys_config       - 系统配置表(键/值/分组/描述)

详细建表SQL见 work/数据库建表脚本.sql

六、API接口设计概要
========================================

认证: POST /api/auth/login, GET /api/auth/profile
巡检点: GET/POST/PUT/DELETE /api/run-points
岗位: GET/POST/PUT/DELETE /api/positions
巡检计划: GET /api/run-plans, POST /api/run-plans/generate-monthly
巡检记录: POST /api/run-records, GET /api/run-records/{id}
设备: GET/POST/PUT /api/equipments, POST /api/equipments/import
设备二维码: GET /api/equipments/{id}/qrcode
检查项: GET/POST /api/equipments/{id}/check-items
检查任务: GET/POST/PUT /api/check-tasks, POST /api/check-tasks/import
检查结果: POST /api/check-results
隐患: POST /api/hazards, PUT /api/hazards/{id}/review/handle/accept
统计: GET /api/stats/daily-run, /api/stats/equipment, /api/stats/hazard
WebSocket: run_progress, hazard_alert, equipment_status, task_complete

========================================
七、NTAG213 NFC标签方案
========================================

推荐NTAG213（成本约0.5-1元/个）
方案: 仅使用UID（硬件唯一标识）
- UID是硬件级别唯一，不可篡改
- 无需担心标签写入寿命
- 成本最低
- 安全性高（手机+服务器双重验证）
